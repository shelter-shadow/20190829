SDL-Linux编译安装

跨平台图形库SDL-Centos7编译安装

简介

SDL库是Simple DirectMedia Layer的缩写，简单直接的多媒体层，不仅包括图像处理，音频处理，输入输出，还支持多线程和事件的开发，而且SDL是跨平台的。因为SDL开源性质，所以非常多的应用都是用SDL作为底层。

下载

在 https://github.com/libsdl-org/SDL/releases/latest 中下载最新SDL适配Linux的 tar.gz 文件。

编译安装

  1. 在Linux中使用 tar -xzvf SDL2-*.*.*.tar.gz 解压下载的压缩包
  2. 进入解压的文件夹
  3. 输入 ./configure 配置安装参数，可以使用 --prefix= 指定希望安装到的目录
  4. 输入 make && make install 完成编译安装
  5. 输入 vim ~/.bashrc 将安装目录下bin文件夹添加进环境变量: export PATH=$PATH:安装目录/bin

liunx循环登录

Linux由于更改环境变量导致登录后黑屏再次进入登录界面解决办法

本来是想一步到位把go语言的bin文件夹添加到环境变量中，但是由于未知原因更改了/etc/environment文件之后系统就再也登录不上了(；д；)于是捣鼓了好久终于把系统救回来了

下面是正篇

1.首先进入单用户模式

  • 重启Linux
  • 在出现Centos Linux(3.10.0......)7和Centos Linux(0-rescure.....)两个选项的界面按e键进入内核编辑模式
  • 找到以LANG=en_US.UTF-8结尾的那一行
  • 将crashkernel前面的ro改为rw
  • 删除rhgb quiet
  • 在此行结尾加上init=/bin/bash
  • 接着按Ctrl+x即可进入单用户模式

2.重置修改过的文件

此时Linux处在DOS命令行的界面下，通过vim或nano打开刚才修改过的文件，将修改内容还原

3.退出单用户模式

  • 输入exec /sbin/init

4.回到用户选择界面登录即可

Linux翻墙

Linux 虚拟机使用物理机代理（以centos7和cjpc为例）

作为一个探头出墙的主要方向，Linux竟然没有几个不错的适配的梯子，下个外国的sample都连接不上。好在作为宿主机的Windows梯子不少，但是物理机和虚拟机之间还有一层可悲的厚障壁，本篇探讨如何让虚拟机上的Linux使用上物理机的vpn。

Windows配置

要让虚拟机能稳定地使用物理机的代理，首先要尽可能的取得详细的物理机代理设置
这里使用Clash for Windows进行代理的部署。(汉化可自行百度)
步骤如下：

  1. 首先使用 Windows键+r 打开运行窗口
  2. 在运行窗口中输入 cmd 打开命令行
  3. 在命令行中输入 ipconfig 查看本机ip配置
  4. 找到无线局域网适配器 WLAN 中的 IPv4 地址并记录，这是本机在局域网中的ip地址
  5. 在使用的梯子官方获取到配置文件或者配置文件网址导入到Clash for Windows中，选择规则代理和合适的节点，然后上外网测试一下，确保Windows能使用vpn翻墙。

Linux配置

  1. 在设置中打开网络设置，找到Net Peoxy选项并打开
  2. 将模式切换为手动
  3. 在下面4个框中输入刚才记录的本机ip地址
  4. ip地址后面输入端口号（端口号在clash主页第一行端口信息处看到，一般为7890）

clash设置

  1. 在clash主页将局域网代理的开关打卡即可

最后返回虚拟机使用自带浏览器打开外网网站，即可正常访问，且命令行内诸如git命令等均能被加速
