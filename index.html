liunx循环登录

Linux由于更改环境变量导致登录后黑屏再次进入登录界面解决办法

本来是想一步到位把go语言的bin文件夹添加到环境变量中，但是由于未知原因更改了/etc/environment文件之后系统就再也登录不上了(；д；)于是捣鼓了好久终于把系统救回来了

下面是正篇

1.首先进入单用户模式

  • 重启Linux
  • 在出现Centos Linux(3.10.0......)7和Centos Linux(0-rescure.....)两个选项的界面按e键进入内核编辑模式
  • 找到以LANG=en_US.UTF-8结尾的那一行
  • 将crashkernel前面的ro改为rw
  • 删除rhgb quiet
  • 在此行结尾加上init=/bin/bash
  • 接着按Ctrl+x即可进入单用户模式

2.重置修改过的文件

此时Linux处在DOS命令行的界面下，通过vim或nano打开刚才修改过的文件，将修改内容还原

3.退出单用户模式

  • 输入exec /sbin/init

4.回到用户选择界面登录即可

Linux翻墙

Linux 虚拟机使用物理机代理（以centos7和cjpc为例）

作为一个探头出墙的主要方向，Linux竟然没有几个不错的适配的梯子，下个外国的sample都连接不上。好在作为宿主机的Windows梯子不少，但是物理机和虚拟机之间还有一层可悲的厚障壁，本篇探讨如何让虚拟机上的Linux使用上物理机的vpn。

Windows配置

要让虚拟机能稳定地使用物理机的代理，首先要尽可能的取得详细的物理机代理设置
这里使用Clash for Windows进行代理的部署。(汉化可自行百度)
步骤如下：

  1. 首先使用 Windows键+r 打开运行窗口
  2. 在运行窗口中输入 cmd 打开命令行
  3. 在命令行中输入 ipconfig 查看本机ip配置
  4. 找到无线局域网适配器 WLAN 中的 IPv4 地址并记录，这是本机在局域网中的ip地址
  5. 在使用的梯子官方获取到配置文件或者配置文件网址导入到Clash for Windows中，选择规则代理和合适的节点，然后上外网测试一下，确保Windows能使用vpn翻墙。

Linux配置

  1. 在设置中打开网络设置，找到Net Peoxy选项并打开
  2. 将模式切换为手动
  3. 在下面4个框中输入刚才记录的本机ip地址
  4. ip地址后面输入端口号（端口号在clash主页第一行端口信息处看到，一般为7890）

clash设置

  1. 在clash主页将局域网代理的开关打卡即可

最后返回虚拟机使用自带浏览器打开外网网站，即可正常访问，且命令行内诸如git命令等均能被加速

建站

Hexo + Github 搭建个人博客

摘要

这个博客就是使用hexo和GitHub进行搭建的，在此记录一下搭建流程以便后来查阅

环境配置

由于博主只有Windows系统，以下是基于Windows的环境配置
所需环境：Git，Node.js， Hexo

GIt配置

  • 在Git的官网上下载Git的安装包
  • Git的安装有许多复杂的选项，这里就直接贴链接了 https://blog.csdn.net/mukes/article/details/115693833
    其中如果选择vim编辑器或者nano编辑器请百度使用方法
  • 一定一定记得勾选添加到环境变量

Node.js配置

  • 官网下载Node.js https://nodejs.org/en/
  • 打开安装包下一步下一步…一定一定记得选上‘Add to Path’
  • 在cmd中使用如下代码查看是否安装成功 如果出现数字版本号则为安装成功

Hexo安装

  • 使用 npm install hexo-cli -g 在本地安装hexo组件

Hexo 建站

  • 打开准备存放博客文件的文件夹，运行cmd

  • 初始化本地博客内容：

    新建完成后目录如下

    _config.yml
    网站的 配置 信息，可以在此配置大部分的参数。

    package.json
    应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，可以自由移除。

    scaffolds
    模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。

    Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

    source

    资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

    themes

    主题 文件夹。Hexo 会根据主题来生成静态页面。

Hexo 写作

  • 使用 hexo new <文章名称> 新建文章
  • 在source文件夹里的_post目录下可以找到新建的文章，格式是 .md
  • 使用文本文档即可编辑，若想要实时预览，可使用vscode等主流IDE及其插件，也可使用网页上的在线编辑，编辑完成后复制进文件即可

Hexo 部署

  • 首先确保你拥有GitHub账号
  • 登录GitHub后新建公开仓库，仓库名为你的GitHub用户名.github,io
  • 打开建好的仓库，将第一个区块的仓库地址（https开头的，例：https://bitbucket.org/JohnSmith/johnsmith.bitbucket.io ）复制下来
  • 修改配置文件
  • 本地试运行：hexo server或者hexo s，将命令行中提供的网址复制到浏览器中回车进入即可看到主页
  • 清除已生成的静态文件：hexo clean
  • 生成静态文件：hexo generate或hexo g
  • 一键部署：hexo deploy或hexo d 注意要先生成静态文件再部署

可能遇到的问题

  1. 首次部署会要求提供GitHub账号密码，请保证网络畅通
  2. 由于墙的不稳定性，有时候会出现 unable to access ... 的错误，一般多提交几次或者开启/关闭梯子即可

摘要

既然是依托于GitHub构建的博客，那么就一定离不开这个广为人知的分布式版本管理系统 Git了

欢迎勘误，联系邮件：ltj161974238@163.com

什么是Git

关于这个网上有太多更加优秀的释义了，推荐百度。

概括来说Git是一个开源的分布式的版本控制系统。

什么是版本控制？

以一个不专业的简单例子来说当你准备进行一次演讲，那么在撰稿阶段你的文件夹里也许会出现诸如:

  • 第一版.docx
  • 第二版.docx
  • 最后一版.docx
  • 最最后一版.docx
  • …

的让人想想就心肺骤停的文件序列。其实每一个这样的docx文件就可以称之为一个版本。而版本控制就是指查找、回退、更新、使用不同版本的文件
单一个文件用这种复制+重命名的方式来进行版本控制往往无伤大雅甚至更加便捷直观，但是对于动辄几十几百个文件的项目来说，再去重复上述步骤我只能敬佩你有毅力。于是版本控制系统呼之欲出，其作用就是像刚才的例子一样存储不同时期不同状态的工程文件。

什么是分布式

分布式的概念太大了，这里只讨论本文中所用的分布式概念。

在此分布式意味着即使是离线状态也可在本地即可完成建立仓库，日志管理，版本管理等一系列版本控制的操作。
